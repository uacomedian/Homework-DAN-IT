Тема: Компоненти архітектури ОС Linux

a) Kernel (ядро)
- Роль: головний компонент ОС, посередник між апаратним забезпеченням і користувацькими програмами.
- Що робить:
  • керує процесами (створення, планування, перемикання контексту);
  • керує пам’яттю (віртуальна пам’ять, сторінки, кеші, mmap);
  • забезпечує доступ до пристроїв через драйвери (диски, мережеві карти, USB тощо);
  • файлові системи (ext4, XFS, btrfs…);
  • мережевий стек (TCP/IP, маршрутизація, фаєрволи через netfilter/nftables);
  • безпека й ізоляція (права доступу, cgroups, namespaces, SELinux/AppArmor).
- Значення: без ядра програми не бачать “заліза”, а система не може гарантувати стабільність, безпеку й ефективний розподіл ресурсів.

b) Libraries (бібліотеки)
- Роль: готові, перевикористовувані блоки коду (динамічні або статичні), які дають стандартні функції та API до системних викликів.
- Приклади: glibc, libm, OpenSSL, libpthread тощо.
- Значення: прискорюють розробку, зменшують розмір програм (спільні .so), спрощують оновлення та підвищують безпеку.

c) System utilities (системні утиліти)
- Роль: інструменти для керування системою та виконання повсякденних задач.
- Приклади: ls/cp/rm, ps/top/kill, ip/ping/ss, apt/dpkg, systemctl/journalctl, useradd/chmod/chown, cron, fdisk/lsblk, df/du.
- Значення: роблять ОС зручною для користувача й адміністратора без прямої взаємодії з ядром.
